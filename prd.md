## 1. 프로젝트 개요

### 1.1 배경 및 목적

**현황 (As-Is)**
- 현재 지급 현황 페이지는 텍스트 위주의 단순 리스트 형태로 구성
- 재단의 성과가 직관적으로 전달되지 않음
- 정적인 페이지로 인한 낮은 사용자 체류 시간
- 모바일 환경에서의 정보 전달력 부족

**목적 (To-Be)**
- 데이터 시각화를 통한 직관적인 정보 전달
- 재단의 투명성 강조 및 신뢰도 향상
- 장학 사업의 사회적 가치를 효과적으로 홍보
- 재단의 브랜드 이미지 제고
- 잠재적 기부자 및 지원자 유치

### 1.2 프로젝트 범위

**포함 사항**
- 현황 페이지 전면 리뉴얼
- 데이터 시각화 대시보드 구현
- 인터랙티브 차트 및 그래프 구현
- 장학생 스토리 섹션 추가
- 연차 보고서 다운로드 기능
- 반응형 디자인 (모바일 최적화)

**제외 사항**
- 기타 페이지 리뉴얼 (1차 범위 외)
- 백엔드 시스템 전면 개편
- 관리자 페이지 개발

## 2. 현황 분석

### 2.1 문제점 분석

| 항목 | 현재 상태 | 문제점 |
|------|-----------|--------|
| **정보 전달** | 연도, 인원, 금액의 단순 텍스트 나열 | 가독성이 낮고 성장의 흐름을 파악하기 어려움 |
| **시각 요소** | 이미지나 차트가 전무함 | 페이지가 정적으로 보이며 방문자의 체류 시간이 짧음 |
| **상호 작용** | 정보 조회 외 기능 없음 | 장학금 신청이나 상세 자료 확인으로의 연결성이 부족함 |
| **모바일 최적화** | 단순 텍스트 위주 | 모바일에서 정보의 중요도가 낮아 보임 |

### 2.2 경쟁사 벤치마킹

**우수 사례**
- 주요 장학재단들의 데이터 시각화 사례 분석
- 인터랙티브 연차 보고서 사례
- 스토리텔링 기반 콘텐츠 구성 사례

## 3. 개선 전략

### 3.1 전략 1: 데이터 시각화 (Data Visualization)

**성장 지표 대시보드**
- 누적 장학생 수를 상단에 큰 폰트로 배치
- 누적 지급액을 강조하여 표시
- 숫자 카운팅 애니메이션 효과 적용
- 실시간 업데이트 느낌의 동적 효과

**추이 그래프**
- 연도별 지급 현황을 시각화
- 막대 그래프 또는 선형 그래프 활용
- 우상향하는 재단의 성장세 강조
- 인터랙티브 호버 효과

### 3.2 전략 2: 정보의 투명성 및 깊이 강화

**상세 통계**
- 학급별(고등/대학) 수혜 비중 파이 차트
- 지역별 지원 범위 시각화
- 분야별 장학금 분포도
- 연도별 비교 분석

**아카이브 연동**
- 해당 연도의 공지사항 링크
- 장학생 선발 공고 바로가기
- 관련 뉴스 및 보도자료 연결
- 포토 갤러리 연동

### 3.3 전략 3: 감성적 스토리텔링 결합

**장학생 한마디**
- 실제 장학생들의 감사 메시지
- 성장 스토리를 슬라이드 형태로 배치
- '숫자' 뒤에 있는 '사람'의 변화 강조
- 사진과 함께 스토리 전달

## 4. 주요 기능 명세

### 4.1 Feature 1: 대화형 차트 (Interactive Charts)

**기능 설명**
- 마우스 호버 시 상세 수치 표시
- 연도별 필터링 기능
- 차트 타입 전환 기능 (막대/선형/파이)
- 데이터 다운로드 기능

**기술 스택**
- Chart.js 또는 Recharts (React 기반)
- D3.js (고급 시각화 필요 시)
- Framer Motion (애니메이션)

**우선순위**: P0 (필수)

**수용 기준 (Acceptance Criteria)**
- [ ] 연도별 장학금 지급액을 막대 그래프로 표시
- [ ] 호버 시 정확한 수치와 비율 표시
- [ ] 연도 범위 필터링 가능 (예: 2020-2024)
- [ ] 모바일에서도 터치 인터랙션 지원
- [ ] 로딩 시 부드러운 애니메이션 효과

### 4.2 Feature 2: 성장 지표 대시보드

**기능 설명**
- 누적 장학생 수 카운팅 애니메이션
- 누적 지급액 카운팅 애니메이션
- 전년 대비 증가율 표시
- 주요 성과 지표 카드

**기술 구현**
- CountUp.js 또는 커스텀 애니메이션
- Intersection Observer API (스크롤 트리거)
- 반응형 그리드 레이아웃

**우선순위**: P0 (필수)

**수용 기준**
- [ ] 페이지 로드 시 0부터 최종 수치까지 카운팅
- [ ] 3개 이상의 핵심 지표 표시
- [ ] 전년 대비 증가율을 %로 표시
- [ ] 모바일에서 카드 형태로 스택 배치
- [ ] 1.5초 이내 애니메이션 완료

### 4.3 Feature 3: 상세 통계 시각화

**기능 설명**
- 학급별(고등/대학) 분포 파이 차트
- 지역별 지원 현황 지도 또는 차트
- 분야별 장학금 분포
- 연도별 비교 분석

**기술 구현**
- Recharts 파이 차트
- 한국 지도 SVG 또는 지도 라이브러리
- 툴팁 및 범례

**우선순위**: P1 (중요)

**수용 기준**
- [ ] 학급별 분포를 파이 차트로 표시
- [ ] 각 섹션 클릭 시 상세 정보 모달 표시
- [ ] 지역별 데이터를 시각적으로 표현
- [ ] 색상 코딩으로 직관적 이해 지원
- [ ] 범례 및 설명 포함

### 4.4 Feature 4: 장학생 스토리 섹션

**기능 설명**
- 장학생 감사 메시지 슬라이더
- 사진과 함께 스토리 전달
- 자동 재생 및 수동 네비게이션
- 랜덤 또는 최신순 정렬

**기술 구현**
- Swiper.js 또는 Embla Carousel
- 이미지 최적화 (Next.js Image)
- 자동 재생 타이머

**우선순위**: P1 (중요)

**수용 기준**
- [ ] 최소 5개 이상의 스토리 표시
- [ ] 자동 재생 (5초 간격)
- [ ] 좌우 네비게이션 버튼
- [ ] 페이지네이션 인디케이터
- [ ] 모바일에서 스와이프 제스처 지원

### 4.5 Feature 5: 퀵 액션 버튼 (CTA)

**기능 설명**
- 장학금 지원 자격 확인 버튼
- 후원 안내 바로가기 버튼
- 연차 보고서 다운로드 버튼
- 문의하기 버튼

**기술 구현**
- 고정 위치 또는 섹션 하단 배치
- 호버 효과 및 애니메이션
- 외부 링크 또는 내부 페이지 연결

**우선순위**: P0 (필수)

**수용 기준**
- [ ] 페이지 하단에 CTA 버튼 배치
- [ ] 각 버튼의 목적이 명확히 전달됨
- [ ] 호버 시 시각적 피드백 제공
- [ ] 모바일에서 터치하기 쉬운 크기 (최소 44x44px)
- [ ] 접근성 기준 준수 (ARIA 레이블)

### 4.6 Feature 6: 연차 보고서 다운로드

**기능 설명**
- 연도별 보고서 목록 표시
- PDF 파일 다운로드 기능
- 파일 크기 및 페이지 수 표시
- 미리보기 기능 (선택사항)

**기술 구현**
- 파일 다운로드 링크
- PDF 뷰어 (선택사항)
- 파일 메타데이터 표시

**우선순위**: P1 (중요)

**수용 기준**
- [ ] 최근 3년간의 보고서 제공
- [ ] 다운로드 버튼 클릭 시 즉시 다운로드
- [ ] 파일명에 연도 포함 (예: SJSF_Report_2024.pdf)
- [ ] 파일 크기 표시
- [ ] 다운로드 실패 시 에러 메시지 표시

### 4.7 Feature 7: 반응형 디자인

**기능 설명**
- 모바일, 태블릿, 데스크톱 최적화
- 터치 제스처 지원
- 적응형 레이아웃
- 성능 최적화

**기술 구현**
- Tailwind CSS 반응형 유틸리티
- CSS Grid 및 Flexbox
- 이미지 최적화
- 레이지 로딩

**우선순위**: P0 (필수)

**수용 기준**
- [ ] 모바일 (< 768px) 레이아웃 최적화
- [ ] 태블릿 (768px - 1024px) 레이아웃 최적화
- [ ] 데스크톱 (> 1024px) 레이아웃 최적화
- [ ] 모든 차트가 반응형으로 동작
- [ ] 터치 디바이스에서 모든 인터랙션 정상 작동

## 6. 디자인 시스템

### 6.1 컬러 팔레트

**Primary Colors**
- Primary: `#1E40AF` (재단 메인 컬러 - 신뢰감)
- Secondary: `#10B981` (성장/긍정 - 차트 강조)
- Accent: `#F59E0B` (주목/행동 유도)

**Neutral Colors**
- Background: `#F9FAFB`
- Surface: `#FFFFFF`
- Text Primary: `#111827`
- Text Secondary: `#6B7280`

**Semantic Colors**
- Success: `#10B981`
- Warning: `#F59E0B`
- Error: `#EF4444`
- Info: `#3B82F6`

### 6.2 타이포그래피

**폰트 패밀리**
- 한글: Pretendard (또는 Noto Sans KR)
- 영문/숫자: Inter

**폰트 크기 스케일**
- Heading 1: 48px (모바일: 32px)
- Heading 2: 36px (모바일: 28px)
- Heading 3: 24px (모바일: 20px)
- Body: 16px
- Caption: 14px

### 6.3 간격 시스템

- Tailwind 기본 스케일 사용 (4px 단위)
- 섹션 간격: 80px (모바일: 48px)
- 컴포넌트 간격: 24px (모바일: 16px)

### 6.4 애니메이션 원칙

- **Duration**: 200-500ms (빠른 피드백), 500-1000ms (페이지 전환)
- **Easing**: ease-in-out (자연스러운 움직임)
- **Trigger**: Intersection Observer (스크롤 기반)

├── Trend Chart Section (추이 차트 섹션)
│   ├── 연도별 지급 현황 그래프
